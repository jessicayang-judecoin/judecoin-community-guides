# Community Guide: Judecoin Service Node Staking & Operations

This is the English version of the community guide above.

It is intended to help international users understand the basic Judecoin Service Node staking and operation process.

This guide is shared as a community operation reference. It is not an official document yet, but I hope it can help new users and provide practical feedback for future documentation.

## 【Overview】

Judecoin Service Node is an important infrastructure component of the Judecoin network. Users can support the network and receive block rewards by locking **23,600 JUDE** and running a server.

Basic process:

Server setup → Start node → Blockchain synchronization → Generate registration command → Submit staking transaction from wallet → Check Service Node public key → Back up Service Node private key → Verify through block explorer

## 【1. Server Preparation】

Each Service Node is recommended to run on an independent server.

Basic recommendations:

- Operating system: Ubuntu 20.04 / 22.04
- Memory: 4GB minimum, 8GB recommended
- Storage: 40GB SSD or above
- Ports: 22 / 16060 / 16063

Required ports:

- 22: SSH remote connection
- 16060: Blockchain synchronization
- 16063: Service Node communication

## 【2. Install Judecoin CLI】

Enter the server terminal and run:

cd ~
sudo apt update
sudo apt install wget tar bzip2 curl -y
wget https://www.judecoin.io/storage/files/cli/judecoin-linux-x64-v3.1.2.tar.bz2
tar -jxvf judecoin-linux-x64-v3.1.2.tar.bz2
cd judecoin-x86_64-linux-gnu-v3.1.2
./judecoind --version

If the version shows `v3.1.2`, the CLI installation is successful.

## 【3. Start the Service Node】

Run the following command on the server:

./judecoind --service-node --service-node-public-ip $(curl -s ifconfig.me) --no-zmq --detach

This command automatically gets the server public IP and starts the node in the background.

Check node running and synchronization status:

./judecoind status

Wait until blockchain synchronization is completed before continuing with registration.

## 【4. Generate the Registration Command】

After synchronization is completed, run the following command on the server:

./judecoind prepare_registration

The system will ask for the operator wallet address.

Notes:

- Use the Primary Address from the CLI wallet
- Do not use a Subaddress
- For an independent Service Node, enter: `23600`
- For a shared node, follow the prompts and enter the corresponding contribution amount and participant addresses

After completing the prompts, the server will generate a registration command similar to:

register_service_node xxxxxxxxxxxxx

This command must be copied completely. Do not modify or omit any part of it.

## 【5. Submit the Staking Transaction from the Wallet】

Open the local CLI wallet:

./judecoin-wallet-cli

If you need to refresh the wallet balance, run:

refresh

Check wallet balance:

balance

After confirming that the wallet has enough balance, paste the complete `register_service_node` command generated by the server into the CLI wallet and execute it.

When the wallet asks for transaction confirmation, enter:

Y

After confirmation, the staking transaction will be submitted to the blockchain.

## 【6. Check Public Key and Back Up Private Key】

Check the Service Node public key:

./judecoind print_sn_key

Back up the Service Node private key:

./judecoin-sn-keys show /home/ubuntu/.bitjudecoin/key_ed25519

This step is very important.

A Service Node has its own node identity private key. If the server is damaged, lost, reinstalled, or migrated, this key may be required to restore the original node identity.

If this key is not backed up, the node identity may be lost and the node may need to be registered again.

## 【7. Verify Through the Block Explorer】

Open the block explorer:

https://www.judeblock.org

Search for the Service Node public key.

If the node can be found in the block explorer, it means the Service Node has been successfully registered.

## 【Security Reminders】

Please pay attention to the following security rules:

- Do not share wallet seed phrases
- Do not share wallet private keys
- Do not share view keys or spend keys
- Do not share wallet files
- Do not share the Service Node private key
- Do not share `.pem` files
- Do not expose server IP addresses, login information, or backend screenshots
- Do not click links from unknown emails
- Do not send seed phrases, private keys, verification codes, or wallet files to anyone
- Use a hardware encrypted USB drive for important backups
- Keep at least two offline backups in separate safe locations
